# Style Guide

Use restrained publication-oriented styling. The diagram should remain clear when scaled down.

## Fonts

- English paper figures: Arial, Helvetica, or Calibri.
- Chinese figures: Microsoft YaHei or SimSun if required by thesis style.
- Default node text: 8-10 pt for paper figures, 12-16 pt for slides.
- Avoid mixed fonts unless bilingual text requires it.

## Colors

Use 3-5 semantic colors. Avoid decorative gradients.

Suggested palette:

- Data/input: `#E8F3E8`
- Model/core: `#EAF2FF`
- Attention/innovation: `#FFF3D6`
- Output/metric: `#FCE8E6`
- Neutral container: `#F6F7F8`
- Line/text: `#2F3437`

For black-white friendly diagrams, distinguish groups by line style, labels, and light fill values rather than hue alone.

## Lines and Arrows

- Main connectors: 0.8-1.2 pt.
- Container borders: 0.6-0.8 pt.
- Use one arrowhead style across the whole figure.
- Avoid crossing connectors; if unavoidable, route one connector with elbows.

## Layout

- Align same-level modules to a grid.
- Keep same-level blocks the same size unless semantic weight differs.
- Use at least 4 mm inner padding for node text.
- Use 8-12 mm page margins for standalone paper figures.
- Leave enough whitespace around labels and arrowheads.

## Text

- Prefer short noun phrases in nodes.
- Put implementation detail in annotations, not inside crowded blocks.
- Use line breaks intentionally for long labels.
- Avoid paragraphs inside nodes.

## Publication Checks

- Single-column paper width is usually around 80-90 mm.
- Double-column paper width is usually around 170-180 mm.
- Slide diagrams can use larger text and wider spacing.
- Export at high resolution for preview and paper insertion.
